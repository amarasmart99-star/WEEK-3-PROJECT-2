# Week 3 Project — Sales Data SQL Analysis

## Overview
This project is part of the AnalystLab Africa Data Analytics Internship Program (Week 3: SQL & Data Querying). It uses SQL to extract, transform, and analyze a sales transactions dataset, covering core querying, advanced SQL concepts, and business-driven analysis.

## Dataset
**Sample Sales Data** (Kaggle)
Source: https://www.kaggle.com/datasets/kyanyoga/sample-sales-data

The dataset contains order-level sales transactions, including product line, sales amount, order date, customer details, country, deal size, and order status.

## Tools Used
- Microsoft SQL Server
- SQL Server Management Studio (SSMS)

## SQL Concepts Covered
- SELECT, WHERE, ORDER BY
- GROUP BY, HAVING, aggregate functions (SUM, AVG, COUNT)
- INNER JOIN
- Subqueries
- Window functions (RANK, ROW_NUMBER, PARTITION BY)

## Files in This Repository
| File | Description |
|------|--------------|
| `Analyst Sales Sql.sql` | Full SQL script containing all 18 queries, from core querying through advanced business analysis |
| `SALES DOCUMENT.docx` | Insights report explaining the purpose and findings of each query |

## Key Insights
- Classic Cars is the top-performing product line, generating over $3.9M in revenue
- Euro Shopping Channel is the highest-spending customer
- Revenue grew from $3.5M (2003) to $4.7M (2004)
- 93% of all orders were successfully shipped
- Medium-sized deals generated the highest total revenue across the business
